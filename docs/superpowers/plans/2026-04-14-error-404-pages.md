# Error & 404 Pages Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add custom branded 404 and error pages in Next.js App Router — `not-found.tsx`, `error.tsx`, and `global-error.tsx` — with bilingual copy (EN/PT-PT) and brand-consistent design.

**Architecture:** `not-found.tsx` and `error.tsx` live inside `app/[language]/(routes)/` so they inherit the full site layout (Header, Footer, fonts, ThemeProvider). `global-error.tsx` lives at root as a last-resort fallback. Language detection for `not-found.tsx` (a server component with no `params` prop) uses a custom `x-language` request header set in middleware. `error.tsx` detects language client-side via `usePathname()`.

**Tech Stack:** Next.js 15 App Router, TypeScript, Tailwind CSS v4 (OKLCH tokens), Vitest (build check only — no DOM environment configured)

## Task 1: Middleware — forward locale as request header

**Files:**
- Modify: `middleware.ts`

The `not-found.tsx` server component receives no `params` prop (Next.js design). Language is passed via a custom `x-language` request header — the documented pattern (see `not-found.mdx` official docs). Must be set on `NextResponse.next({ request: { headers } })`, NOT on `response.headers.set()` — the latter is not readable by server components via `headers()`.

The middleware has three exit paths where a page is actually rendered (not redirected). Both need the header.

- [ ] **Step 1: Update `middleware.ts` — full file replacement**

```typescript
import type { NextRequest } from "next/server";
import { NextResponse } from "next/server";
import { match } from "@formatjs/intl-localematcher";
import Negotiator from "negotiator";

const locales = ["en", "pt"];
const defaultLocale = "pt";

function getLocale(request: NextRequest) {
  const acceptedLanguage = request.headers.get("accept-language") ?? "pt";
  const headers = { "accept-language": acceptedLanguage };
  const languages = new Negotiator({ headers }).languages();
  return match(languages, locales, defaultLocale);
}

export function middleware(request: NextRequest) {
  const pathname = request.nextUrl.pathname;

  // ✅ BAIL EARLY: Allow non-localized pages /bio and /card
  if (pathname === "/bio" || pathname === "/card") {
    return NextResponse.next();
  }

  // ✅ BAIL EARLY: Static assets
  if (
    /\.(svg|png|jpg|jpeg|gif|ico|webp|css|js|map|json|txt|xml|woff2?|ttf)$/.test(
      pathname
    )
  ) {
    return NextResponse.next();
  }

  // Determine if the pathname is missing a locale
  const pathnameIsMissingLocale = locales.every(
    (locale) => !pathname.startsWith(`/${locale}/`) && pathname !== `/${locale}`
  );

  if (pathnameIsMissingLocale) {
    const locale = getLocale(request);
    // Redirect — no page is rendered here, header not needed
    return NextResponse.redirect(new URL(`/${locale}${pathname}`, request.url));
  }

  // Rewrite /pt/projetos(/*) → /pt/projects(/*) — locale is always "pt" here
  const projectsRewrite = pathname.match(/^\/(pt)\/projetos(\/.*)?$/);
  if (projectsRewrite) {
    const rest = projectsRewrite[2] || "";
    const requestHeaders = new Headers(request.headers);
    requestHeaders.set("x-language", "pt");
    return NextResponse.rewrite(
      new URL(`/pt/projects${rest}`, request.url),
      { request: { headers: requestHeaders } }
    );
  }

  // Locale is present in pathname — extract it and pass through
  const locale =
    locales.find(
      (l) => pathname.startsWith(`/${l}/`) || pathname === `/${l}`
    ) ?? defaultLocale;
  const requestHeaders = new Headers(request.headers);
  requestHeaders.set("x-language", locale);
  return NextResponse.next({ request: { headers: requestHeaders } });
}

export const config = {
  matcher: [
    "/((?!api|_next|.*\\..*|admin|sitemap.xml|webhook|sentry-example-page|bio|card).*)",
  ],
};
```

## Task 2: 404 page — `not-found.tsx`

**Files:**
- Create: `app/[language]/(routes)/not-found.tsx`

Server component. Placed inside `[language]/(routes)/` so it inherits the full layout (Header, Footer, fonts, ThemeProvider) from `app/[language]/(routes)/layout.tsx`. Reads `x-language` from request headers (set in Task 1). Falls back to `'pt'` if header is absent.

Visual design: left-aligned, large ghost "404" in terracotta-light positioned absolutely behind the headline. Headline overlaps via `z-index`. Body text and two CTAs below.

- [ ] **Step 1: Create `app/[language]/(routes)/not-found.tsx`**

```typescript
import { headers } from "next/headers";
import type { Metadata } from "next";
import Link from "next/link";

export const metadata: Metadata = {
  robots: { index: false, follow: false },
};

const copy = {
  en: {
    headline: "This page doesn't exist.",
    subtext:
      "You may have followed a broken link or mistyped the address. Here's where to go instead.",
    primary: "Back to home",
    secondary: "View my work",
  },
  pt: {
    headline: "Esta página não existe.",
    subtext:
      "Pode ter seguido um link antigo ou introduzido o endereço errado. Volta ao início ou explora o meu trabalho.",
    primary: "Voltar ao início",
    secondary: "Ver o meu trabalho",
  },
} as const;

export default async function NotFound() {
  const headersList = await headers();
  const lang = headersList.get("x-language") === "en" ? "en" : "pt";
  const t = copy[lang];

  return (
    <main className="flex min-h-[60vh] items-center px-4 py-24 md:px-8">
      <div className="mx-auto w-full max-w-[1200px]">
        <div className="relative inline-block">
          {/* Ghost 404 — decorative, sits behind headline */}
          <span
            aria-hidden="true"
            className="pointer-events-none absolute -left-1 -top-3 z-0 select-none font-heading text-[9rem] font-extrabold leading-none tracking-tighter text-[var(--color-brand-light)] opacity-55 md:text-[11rem]"
          >
            404
          </span>
          {/* Content overlaps ghost number via z-index */}
          <div className="relative z-10 pt-14 md:pt-16">
            <h1 className="mb-4 font-heading text-3xl font-semibold leading-snug text-[var(--color-text)] md:text-4xl">
              {t.headline}
            </h1>
            <p className="mb-8 max-w-[480px] text-base leading-relaxed text-[var(--color-text-muted)]">
              {t.subtext}
            </p>
            <div className="flex flex-wrap items-center gap-4">
              <Link
                href={`/${lang}`}
                className="inline-flex items-center gap-2 rounded-md bg-[var(--color-brand)] px-7 py-3.5 text-base font-semibold text-white transition-all hover:bg-[var(--color-brand-dark)] hover:-translate-y-px dark:text-[var(--color-surface)]"
              >
                {t.primary}
              </Link>
              <Link
                href={`/${lang}/projects`}
                className="text-base text-[var(--color-text-muted)] underline underline-offset-4 transition-colors hover:text-[var(--color-text)]"
              >
                {t.secondary} →
              </Link>
            </div>
          </div>
        </div>
      </div>
    </main>
  );
}
```

## Task 3: Error page — `error.tsx`

**Files:**
- Create: `app/[language]/(routes)/error.tsx`

Client component (Next.js requirement for error boundaries). Inherits the full layout. Language detected via `usePathname()` — parses first path segment. Props: `{ error, reset }` where `reset()` clears error state for recoverable errors. Home link provides exit for unrecoverable server errors.

Same visual language as 404 — same font stack and token usage — but no ghost number.

- [ ] **Step 1: Create `app/[language]/(routes)/error.tsx`**

```typescript

import Link from "next/link";
```

## Task 4: Global error page — `global-error.tsx`

**Files:**
- Create: `app/global-error.tsx`

Last-resort fallback. Catches errors thrown in the root layout (ThemeProvider, fonts, etc.) — rare in production. **Must** include its own `<html>` and `<body>` — it replaces the entire page. No layout context available so no CSS tokens, no next/font. Uses hardcoded hex values. English only (language detection not feasible without layout context). Uses `unstable_retry` (the correct prop for `global-error.tsx` per Next.js docs — distinct from `reset` used in segment-level `error.tsx`).

## Notes

- **`global-error.tsx` is hard to trigger manually** — it only fires when the root layout itself throws. Trust the TypeScript check + `npm run build` for this one.
- **Dark mode** on `not-found.tsx` and `error.tsx` works automatically — both inherit the ThemeProvider from the layout and use CSS custom properties (`var(--color-*)`) that dark mode overrides.
- **`unstable_retry` vs `reset`:** `global-error.tsx` uses `unstable_retry` (re-fetches AND re-renders). Segment-level `error.tsx` uses `reset` (clears error state). These are different and not interchangeable.
